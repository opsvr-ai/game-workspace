> **状态**: 设计中
> **基于**: 2026-06-25 代码审查报告

---

## 1. 策略概述

### 1.1 当前完成度：~55%

| 层 | 完成度 | 评估 |
|----|--------|------|
| 后端 API | 80% | 核心逻辑完备，缺校验和边界处理 |
| 前端页面 | 35% | 仅 3/14 页面可用，10 个占位页 |

每阶段后端加固 + 前端补全同步进行，以"可运行的功能增量"为里程碑。

---

## 2. Phase 2 — 核心业务闭环（预计 10-12 天）

> **目标**：客服 ↔ 陪玩全链路可用。CS 创建订单 → 入池/指定 → 陪玩抢单/接收 → 确认 → 完成。

### 2.1 后端加固

| 任务 | 优先级 | 说明 |
|------|--------|------|
| DTO 校验 | P0 | Orders/Customers/Billing 加 class-validator |
| 订单状态机校验 | P0 | PENDING→GRABBED→CONFIRMED→DONE 严格流转 |
| WebSocket 集成 | P0 | WsGateway.pushOrder/broadcastToStudio 接入 OrdersService |
| 统一错误处理 | P1 | GlobalExceptionFilter + 中文错误消息 |
| 日志系统 | P1 | 关键操作记录（登录、派单、报账审核） |
| API 文档 | P2 | Swagger/OpenAPI 自动生成 |

### 2.2 前端补全（Phase 2 重点）

**CS 角色（优先 — 日常使用频率最高）：**
- ✅ 派单工作台（已完成）
- [ ] 派单记录页（订单历史、状态筛选）
- ✅ 陪玩状态（已完成）

**ADMIN 角色：**
- [ ] 派单管理（全订单视图、手动干预）
- [ ] 陪玩管理（列表、状态、收入预览）
- [ ] 客户管理（复用 owner 组件，增加归属调整）
- [ ] 报账审核（pending/rejected/approved 三栏）
- [ ] 收入流水（日/月切换）

**OWNER 角色：**
- [ ] 员工管理（列表、角色分配、密码重置）
- [ ] 工作室管理（CRUD）
- [ ] 客户端授权（审批注册申请）

### 2.3 数据持久化

- PostgreSQL + Redis 通过 Docker Compose 管理
- 数据目录：`docker/data/postgres/` 和 `docker/data/redis/`
- 容器自动重启（`restart: unless-stopped`）

## 3. Phase 3 — 报账统计（预计 5-7 天）

> **目标**：老板/管理员可见全链路财务数据。

### 3.1 后端

- 截图上传接口（Multer + 本地存储 `uploads/screenshots/`）
- 报账批量审核
- 日/月营收导出（CSV）
- 盈亏统计增强（按类型/按陪玩/按客户多维度）

### 3.2 前端

- [ ] OWNER 盈亏统计页（折线图 + 饼图，需二级密码）
- [ ] ADMIN 报账审核页（截图预览 + 批量操作）
- [ ] 数据导出按钮

---

## 4. Phase 4 — Agent + 远程控制（预计 5-7 天）

> **目标**：陪玩桌面端可用，远程控制可靠。

### 4.1 Go Agent

- Linux 网络限速（`tc` + `netlink` 替代 PowerShell）
- Linux 系统命令（`systemctl poweroff` 替代 `shutdown`）
- WebUI 完成（工作模式/娱乐模式切换 + 计时显示 + 订单接收弹窗）

### 4.2 前端

- [ ] ADMIN 远程控制页（选择陪玩 → 发送关机/重启/限速命令）
- [ ] 命令执行历史记录

### 4.3 WebSocket

- 命令确认超时处理
- 心跳断开告警

---

## 5. 跨阶段约束

### 5.1 代码规范

- 提交遵循 Conventional Commits（`feat:` / `fix:` / `chore:`）
- 每完成一个功能文件后提交，而非批量提交
- 自动更新 CHANGELOG.md

### 5.2 环境变量

```env
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/chunlv
REDIS_URL=redis://localhost:6379
JWT_SECRET=dev-secret-change-in-production
JWT_REFRESH_SECRET=dev-refresh-secret-change-in-production
```

### 5.3 不在此次范围

- CI/CD 管道
- 单元测试/E2E 测试（Phase 4 之后统一补）
- 生产部署配置
- 第三方登录集成
