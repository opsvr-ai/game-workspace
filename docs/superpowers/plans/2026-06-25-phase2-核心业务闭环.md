# Phase 2 — 核心业务闭环实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 打通客服↔陪玩全链路。CS 创建订单 → 入池/指定 → 陪玩抢单/接收 → 确认 → 完成，所有角色页面可用。

**Architecture:** 后端加固（DTO class-validator + 订单状态机 + WebSocket 事件推送）→ 前端 11 个占位页面逐角色实现。每任务独立可测。

**Tech Stack:** Nest.js 10 + Prisma + class-validator, React 18 + Ant Design 5 + Zustand, Socket.IO, PostgreSQL 16 + Redis 7

## Global Constraints

- Conventional Commits（feat:/fix:/chore:），每任务完成后 git commit
- JWT 双 Token（access 15min / refresh 7d），4 角色 RBAC
- API 响应格式 `{ code, message, data }`
- 前端路由：`/login`, `/owner/*`, `/admin/*`, `/cs/*`
- 所有中文 UI / 错误消息
- Docker Compose 管理数据库（`docker compose -f docker/docker-compose.yaml up -d`）

## 后端任务（3 个）

### Task 1: Order DTO Validation + State Machine

**Files:**
- Create: `apps/server/src/orders/dto/create-order.dto.ts`
- Modify: `apps/server/src/orders/orders.service.ts`
- Modify: `apps/server/src/orders/orders.controller.ts`

**DTO** — `CreateOrderDto` with class-validator decorators: `@IsEnum` for type/dispatchType, `@IsString` for studioId/customerId/gameName, `@IsNumber @Min(0)` for amount, `@IsBoolean` for isOnline, `@IsOptional` for duration/companionId/customFields.

**State Machine** — Add `VALID_TRANSITIONS` map to `OrdersService`:
```
PENDING → [GRABBED, CANCELLED]
GRABBED → [CONFIRMED, CANCELLED]
CONFIRMED → [DONE, CANCELLED]
```
Add `validateTransition(order, targetStatus)` private method. Call it at start of `grab()`, `confirm()`, `complete()`, `cancel()`.

**Controller** — Change `@Body() dto: any` to `@Body() dto: CreateOrderDto` in `create()`.

- [ ] Create DTO file
- [ ] Add state machine validation to OrdersService
- [ ] Update controller to use typed DTO
- [ ] Commit `feat(orders): add DTO validation and state machine`

### Task 2: Global ValidationPipe + ExceptionFilter

**Files:**
- Modify: `apps/server/src/main.ts`
- Create: `apps/server/src/common/http-exception.filter.ts`

**ValidationPipe** — Add `app.useGlobalPipes(new ValidationPipe({ whitelist: true, transform: true }))` to `main.ts`. This enables all class-validator DTOs globally.

**ExceptionFilter** — Create `HttpExceptionFilter` implementing `ExceptionFilter`. Catch `BadRequestException` (validation errors) → return `{ code: 400, message: formattedErrors }`. Catch generic `HttpException` → return `{ code: status, message: exception.message }`.

- [ ] Add global ValidationPipe to main.ts
- [ ] Create common exception filter with Chinese-friendly messages
- [ ] Register filter in main.ts: `app.useGlobalFilters(new HttpExceptionFilter())`
- [ ] Commit `feat(server): add global validation pipe and exception filter`

### Task 3: WebSocket Integration with OrdersService

**Files:**
- Modify: `apps/server/src/orders/orders.module.ts`
- Modify: `apps/server/src/orders/orders.service.ts`

**Module** — Import `WsModule` into `OrdersModule` so `WsGateway` is injectable.

**Service** — Inject `WsGateway` into `OrdersService`. After `create()`: call `this.wsGateway.broadcastToStudio(studioId, 'order:pool_updated', newOrder)`. After `assign()`: call `this.wsGateway.pushOrder(companionId, updatedOrder)`. After `grab()`: call `this.wsGateway.broadcastToStudio(studioId, 'order:pool_updated', ...)` to notify CS the pool changed.

- [ ] Import WsModule in OrdersModule
- [ ] Inject WsGateway into OrdersService
- [ ] Add push events in create/assign/grab methods
- [ ] Commit `feat(orders): integrate WebSocket push for real-time order updates`
